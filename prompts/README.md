# CoderPad-Style Drills

These prompts are intentionally thinner than the API-specific project folders.

Use them to practice the live-coding version of the same skills: read a short problem, clarify requirements, implement the smallest working client behavior, and add a few tests.

Suggested use:

1. Pick one prompt.
2. Timebox yourself to 30–45 minutes.
3. Start from a blank file or the minimal signature in the prompt.
4. Write tests as you go, using mocked HTTP responses.
5. Stop when the acceptance criteria pass, not when the client feels complete.

The prompts avoid full project scaffolding on purpose. In a real screen, the interviewer usually wants to watch how you structure the problem, not how polished the repo is.
